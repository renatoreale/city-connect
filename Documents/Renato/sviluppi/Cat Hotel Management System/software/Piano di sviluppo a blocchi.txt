Premessa tecnica (vincoli non negoziabili)

L’applicazione dovrà integrarsi con un database MySQL già esistente.

Si specifica che:

È consentita la creazione di nuove tabelle.

Le tabelle MySQL già esistenti non devono essere modificate in modo distruttivo.

È ammesso il mapping applicativo verso le tabelle esistenti (ORM o equivalente), purché non vengano alterati dati o strutture già in uso.

Non vengono imposti vincoli su:

linguaggi di programmazione

framework backend o frontend

architettura applicativa (monolite o servizi)

provider email o librerie PDF
a condizione che tutti i requisiti funzionali descritti nel presente documento siano rispettati.

Le DDL delle tabelle MySQL esistenti verranno fornite in una fase successiva, dopo l’allineamento sui requisiti funzionali e prima della definizione finale del data model e dell’implementazione.

BLOCCO 0 — Fondazioni tecniche

Obiettivo: predisporre una base solida, sicura e auditabile su cui costruire tutte le funzionalità successive.

Contenuto

Setup progetto applicativo

Connessione a database MySQL (con rispetto dei vincoli sopra indicati)

Autenticazione utenti

Sistema di ruoli e permessi (RBAC)

Multi-tenant:

entità “Pensione / Tenant”

segregazione dei dati per tenant

Audit append-only:

tracciamento create / update / delete

registrazione di utente, ruolo, tenant e timestamp

Output

Login funzionante

Utenti associabili a uno o più tenant

Isolamento dati garantito

Audit popolato automaticamente

Criteri di accettazione

Nessun utente può accedere a dati di tenant non autorizzati

Ogni modifica a entità funzionali produce una riga di audit

L’audit non è modificabile né cancellabile
